# Backlog

> **Inbox for new ideas.** Anything that pops up mid-build goes here, with date and a one-line context. Nothing in this file is a commitment. Items are evaluated only at phase-shift checkpoints, never mid-phase.

## Rules

1. **No additions to the active plan until Phase 1 ships.** New ideas go here, not into the roadmap.
2. **Capture, don't elaborate.** One line is enough. Polishing happens later.
3. **Review at phase boundaries.** When a phase ends, items here are either promoted to a real plan, moved to `FUTURE_FEATURES.md`, or deleted.
4. **Killing items is a feature.** Most ideas don't survive review. That's the point.

## Format
[YYYY-MM-DD] Idea — one-line context why it came up
