# Future Features

> **Deliberately deferred features.** These have been evaluated and parked — they are *not* in the active plan, but they are not dead either. Each item has a reason for parking and a possible revisit window.

## Difference from `BACKLOG.md`

- `BACKLOG.md` is the **inbox** — random ideas waiting for evaluation.
- `FUTURE_FEATURES.md` is the **parking lot** — features that survived initial review but were postponed for strategic reasons (scope, timing, dependency on data, user demand signals, etc.).

## Items

### Inverse dupes for cheap fragrances
**Parked:** April 2026
**Reason:** Most fragrance dupe content moves cheap → expensive ("this $30 bottle smells like a $300 niche"). The reverse — "this $300 niche actually smells like the $30 budget thing you already own" — is unexplored and high-signal, but requires a much larger fragrance database than Phase 1 has. Revisit after Phase 2 data audit.

### Influencer collections
**Parked:** April 2026
**Reason:** A page where notable fragrance reviewers' personal collections are displayed and discussed. Requires permission, scraping policy review, and a content pipeline. Not a Phase 1 priority. Revisit during Phase 4 if useful for FragBench narrative.

### Social recommendations
**Parked:** April 2026
**Reason:** "People who own X also love Y" — collaborative filtering across users. Requires meaningful user count to function, which Phase 1 won't have. Mathematically valid only at scale. Revisit post-launch if user count crosses ~100.

### Paid tier
**Parked:** April 2026
**Reason:** Monetization is a distraction from the portfolio thesis. FragBro is a graduate school portfolio piece, not a startup. If a paid tier ever makes sense, it's post-2028, post-admissions.
